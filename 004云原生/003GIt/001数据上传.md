假设你想把 `D:/笔记/运维学习/` 这个文件夹推送到 GitHub。

------

### **1️⃣ 生成 SSH 密钥（只需一次）**

如果你还没生成过：

```
ssh-keygen -t rsa -b 4096 -C "你的GitHub邮箱"
```

一路回车即可。

生成后，会在 `~/.ssh/` 下看到：

```
id_rsa      （私钥）
id_rsa.pub  （公钥）
```

------

### **2️⃣ 把公钥添加到 GitHub**

复制公钥内容：

```
cat ~/.ssh/id_rsa.pub
```

然后：

> GitHub → 右上角头像 → **Settings** → **SSH and GPG keys** →
>  点击 **New SSH key** → 粘贴内容 → 保存。

------

### **3️⃣ 添加 GitHub 主机信任**

```
ssh-keyscan github.com >> ~/.ssh/known_hosts
```

------

### **4️⃣ 测试 SSH 连接**

```
ssh -T git@github.com
```

若出现：

```
Hi yourname! You've successfully authenticated, but GitHub does not provide shell access.
```

✅ SSH 配置成功。

------

### **5️⃣ 初始化本地仓库**

进入你的项目目录：

```
cd /d/笔记/运维学习
git init
```

------

### **6️⃣ 创建并提交文件**

```
echo "# 我的运维学习笔记" > README.md
git add .
git commit -m "first commit"
```

------

### **7️⃣ 添加远程仓库（SSH方式）**

> 注意：要用 SSH 地址，不是 HTTPS 地址！

在你的 GitHub 仓库页复制类似：

```
git@github.com:ohh-123-a/ops-notes.git
```

然后执行：

```
git remote add origin git@github.com:ohh-123-a/ops-notes.git
```

------

### **8️⃣ 推送到 GitHub**

```
git branch -M main
git push -u origin main
```

第一次推送会花一点时间，看到：

```
Enumerating objects...
To github.com:ohh-123-a/ops-notes.git
```

说明推送成功 ✅

------

### **9️⃣ （可选）如果还想推送到 Gitee**

你可以添加第二个远程仓库：

```
git remote add gitee git@gitee.com:chenjw1203/ohh.git
git push -u gitee main
```

这样你一个命令就能推送到两个仓库：

```
git push origin main && git push gitee main
```

------

## ✅ 最终结构

你的 `.ssh/` 文件夹应该像这样：

```
~/.ssh/
├── config           # 可选（存放多仓库配置）
├── id_rsa           # 私钥
├── id_rsa.pub       # 公钥（上传到 GitHub/Gitee）
├── known_hosts      # 已信任的主机列表
```
